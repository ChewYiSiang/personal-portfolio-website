## Overview

Smart Garden Assistant is a full-stack IoT project that creates an autonomous environmental monitoring system. Sensor data flows from ESP32-C3 edge hardware through a streaming pipeline to an AI agent that provides actionable gardening recommendations.

## Hardware Stack

| Component | Role |
|---|---|
| ESP32-C3 | Main microcontroller (Wi-Fi + BLE) |
| DS18B20 | Waterproof soil temperature (1-Wire) |
| Capacitive Moisture Sensor | Soil moisture (ADC) |
| DHT22 | Ambient temperature & humidity |

## Data Pipeline

```
ESP32-C3 → MQTT (QoS 1) → Mosquitto Broker
                                  ↓
                            Telegraf subscriber
                                  ↓
                          InfluxDB time-series DB
                                  ↓
                    Python LangChain AI Agent
                                  ↓
                       Dashboard / Push Alerts
```

## Firmware (C++)

```cpp
#include <Arduino.h>
#include <WiFi.h>
#include <PubSubClient.h>
#include <OneWire.h>
#include <DallasTemperature.h>
#include <ArduinoJson.h>

#define ONE_WIRE_BUS  4
#define MOISTURE_PIN  A0

OneWire oneWire(ONE_WIRE_BUS);
DallasTemperature sensors(&oneWire);

void publishReadings(PubSubClient& client) {
  sensors.requestTemperatures();
  float tempC    = sensors.getTempCByIndex(0);
  int   rawADC   = analogRead(MOISTURE_PIN);
  float moisture = map(rawADC, 4095, 1500, 0, 100);  // calibrated range

  StaticJsonDocument<128> doc;
  doc["temp_c"]       = tempC;
  doc["moisture_pct"] = moisture;
  doc["ts"]           = millis();

  char payload[128];
  serializeJson(doc, payload);
  client.publish("garden/sensors", payload, /*retain=*/false);
}
```

## Sensor Calibration

Moisture percentage is derived via linear interpolation between dry and saturated ADC readings:

$$M\% = \frac{V_{raw} - V_{dry}}{V_{wet} - V_{dry}} \times 100$$

Where $V_{dry}$ and $V_{wet}$ are empirically measured at 0% and 100% saturation.

## AI Recommendation Agent

```python
from langchain.agents import initialize_agent, Tool
from langchain_openai import ChatOpenAI
import influxdb_client

def query_sensor_stats(hours: int = 24) -> str:
    client = influxdb_client.InfluxDBClient(
        url="http://localhost:8086", token="...", org="garden"
    )
    query = f"""
    from(bucket: "sensors")
      |> range(start: -{hours}h)
      |> mean()
    """
    tables = client.query_api().query(query)
    # Format results as a readable string for the agent
    return format_tables(tables)

tools = [
    Tool(
        name="SensorStats",
        func=query_sensor_stats,
        description="Fetch average temperature and moisture for the past N hours."
    )
]

agent = initialize_agent(
    tools,
    ChatOpenAI(model="gpt-4o-mini", temperature=0),
    agent="zero-shot-react-description",
    verbose=True,
)
```

## Key Learnings

- **1-Wire pull-up**: DS18B20 requires a 4.7 kΩ pull-up resistor — omitting it causes intermittent reads
- **QoS 1** (at-least-once delivery) is appropriate for sensor telemetry; QoS 2 adds unnecessary overhead
- **Capacitive sensors** dramatically outlast resistive ones in damp soil — worth the slight extra cost
- **InfluxDB Flux** is powerful but verbose; Telegraf handles the MQTT → InfluxDB bridging cleanly
